# 网页展示模块

## 📋 功能说明

提供可视化界面，展示模块1（榜单数据）和模块2（新上榜产品）的数据。

### 页面列表

- **index.html** - 主入口页面，显示项目概览和模块卡片
- **scraper.html** - 榜单数据页面，展示每日爬取的榜单
- **detector.html** - 新上榜产品页面，展示识别的新产品

---

## 🚀 快速开始

### 1. 启动Web服务器

```bash
# 方式1：使用启动脚本（推荐）
./scripts/start_web.sh

# 方式2：手动启动
cd /path/to/app_moitor
python3 -m http.server 8000
```

### 2. 访问网页

在浏览器中打开：
```
http://localhost:8000/web/
```

### 3. 停止服务器

按 `Ctrl+C` 停止服务器

## 🔧 技术说明

### 前端技术

- **HTML5** - 页面结构
- **CSS3** - 样式设计（渐变、阴影、动画）
- **原生JavaScript** - 交互逻辑，无框架依赖

### 数据加载

使用 `fetch` API 读取本地JSON文件：
```javascript
const data = await fetch('../data/raw/2026-02-12/app_store/health_fitness.json');
```

### CORS问题解决

本地HTML文件无法直接读取本地JSON（浏览器安全限制），因此需要启动HTTP服务器。

## ⚠️ 注意事项

### 1. 必须使用HTTP服务器

**❌ 错误方式**：直接双击HTML文件
```
file:///Users/.../web/index.html
```
会遇到CORS错误，无法加载JSON数据。

**✅ 正确方式**：启动HTTP服务器
```bash
./scripts/start_web.sh
# 访问 http://localhost:8000/web/
```

### 2. 数据文件路径

确保数据文件存在于正确位置：
```
data/
├── raw/
│   └── 2026-02-12/
│       ├── app_store/
│       └── google_play/
├── new_apps/
│   └── 2026-02-12.json
└── analyzed_apps.json
```

### 3. 浏览器兼容性

推荐使用现代浏览器：
- ✅ Chrome 90+
- ✅ Firefox 88+
- ✅ Safari 14+
- ✅ Edge 90+

## 🔍 调试技巧

### 查看控制台

打开浏览器开发者工具（F12），查看：
- Network - 检查文件是否加载成功
- Console - 查看JavaScript错误

### 常见问题

**Q: 页面显示"加载中..."但无数据？**
A: 检查数据文件是否存在，路径是否正确

**Q: 图标无法显示？**
A: 图标URL可能失效，已添加默认占位图

**Q: 日期列表为空？**
A: 确保至少运行过一次爬虫，生成了数据文件
